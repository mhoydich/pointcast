# PointCast v2 End-to-End QA Report

**Prepared by:** Manus (Operations + QA)
**Date:** April 17, 2026
**Target:** [pointcast.xyz](https://pointcast.xyz) (v2)

This report details the findings from a structured end-to-end QA pass on the PointCast v2 site. The testing covered functional, visual, technical, and accessibility aspects of the application.

## 1. High-Priority Issues (Blockers)

### 1.1. Missing 404 Handling
The application fails to return a proper 404 HTTP status code or a dedicated error page for non-existent routes. Navigating to invalid URLs (e.g., `/nonexistent-page` or `/b/9999`) returns a 200 OK status and renders the homepage content. This will cause significant SEO issues as search engines will index duplicate content.

### 1.2. Broken Noun Images (noun.pics)
Across the site, numerous SVG images served from `noun.pics` are failing to render visually, despite returning a 200 OK status. The SVGs contain valid markup, but the specific combination of shapes and colors results in invisible or blank images within the block cards. This affects the visual identity of almost every block.

### 1.3. Sitemap and blocks.json Discrepancy
The `/blocks.json` endpoint reports 40, but the `/sitemap-blocks.xml` contains 51 URLs. The sitemap incorrectly includes non-block routing URLs such as `/c/front-door`, `/for-agents`, and the root `/` path. The sitemap should either be renamed to a general sitemap or strictly filtered to only contain `/b/` block URLs.

## 2. Functional & UX Issues

### 2.1. Footer Copyright Year
The copyright year in the footer is hardcoded to "2026" and also contains a typo/bug where "9024" appears in the text node nearby (`El Segundo · 90245`). The year should ideally be dynamic to prevent manual updates in the future.

### 2.2. Missing H1 Tags
Several pages, including the homepage, lack a primary `<h1>` tag. This is a critical accessibility and SEO omission. The site relies heavily on visual headers (e.g., "POINTCAST" in the nav) but does not semantically structure the document outline correctly.

### 2.3. Custom Order Toggle State
The "CUSTOM ORDER" toggle on the homepage functions, but its state is not persistently saved across hard refreshes. Users who reorder blocks will lose their custom arrangement upon returning to the site later.

### 2.4. About Page Content
The `/about` page currently functions only as a visual navigation hub with links to external profiles (LinkedIn, GitHub, etc.) and internal routes. It lacks actual descriptive text or an article about the author or the project, which contradicts the expectation set by the "About" label.

## 3. Technical & Performance Findings

### 3.1. Third-Party Embed Performance
The site loads Spotify embeds (e.g., `open.spotify.com/embed/playlist/...`) which significantly impact the total page load time. During testing, these iframes took over 6.4 seconds to fully resolve. Consider lazy-loading these embeds or replacing them with lightweight placeholder images that load the iframe only upon user interaction.

### 3.2. Missing CSS Variables
The CSS architecture defines variables like `--color-bg`, `--color-fg`, `--font-mono`, and `--font-sans`, but they are not consistently applied or exposed at the `:root` level, resulting in fallback values being used by the browser.

### 3.3. Accessibility (A11y)
While the site implements basic ARIA labels on block links, it fails to provide `alt` text for 37 out of 58 images on the homepage. Most of these are the broken `noun.pics` SVGs, which should at least have `alt=""` (decorative) if they convey no semantic meaning.

## 4. Recommendations for Claude Code

1.  **Implement strict 404 routing:** Ensure Astro (or the underlying framework) correctly handles unmatched routes and returns a 404 status code with a custom error page.
2.  **Investigate noun.pics rendering:** Verify if the `noun.pics` service has changed its SVG output format or if a CSS rule (e.g., `image-rendering: pixelated`) is interfering with the display.
3.  **Clean up the sitemap:** Separate the general site pages from the block entries, or ensure `sitemap-blocks.xml` only generates `<loc>` entries for items in the `blocks.json` collection.
4.  **Add semantic HTML:** Ensure every page has exactly one `<h1>` tag representing the main topic of that route.
5.  **Optimize embeds:** Implement a facade pattern for Spotify and other heavy third-party iframes to improve initial load performance.
