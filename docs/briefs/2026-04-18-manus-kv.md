# Manus brief — Round 3 · Cloudflare KV bindings

**Audience:** Manus operating on Cloudflare Pages dashboard via computer-use.

**Context:** Three async surfaces shipped today (`/ping`, `/sprint` + `/api/queue`, `/drop`). Each has a Pages Function that env-guards against an unbound KV namespace. Right now all three return `503 { reason: "key-not-bound" }` on POST, with documented fallbacks (`docs/inbox/`, `docs/queue/`, `docs/drops/`). The autonomous cron loop works without these — but binding them lets Mike control cc from his phone via the `/sprint` and `/drop` UIs without committing files.

**Three KV namespaces to create + bind. Atomic. ~20 min total.**

## Notes for Manus

- **Three separate namespaces, not one.** Each KV namespace is a key-space; using one for everything mixes data and complicates retention/cleanup.
- **Each namespace already has TTL hints in the function code:** ping = 90 days, queue = 30 days, drop = 60 days. KV honors `expirationTtl` on `put()` so old entries auto-evict — no maintenance needed.
- **Binding name MUST exactly match** `PC_PING_KV` / `PC_QUEUE_KV` / `PC_DROP_KV`. Typo means the function still returns 503.
- **Apply to Production only first.** Preview deploys can stay unbound — they're for testing UI, not for cc reads.
- **No code change needed.** All three functions already check `env.PC_*_KV` and degrade gracefully. Bind once, redeploy once, all three live.

## Why this matters

Without bindings:
- `/sprint` PICK shows "KV not bound — see fallback" message; Mike has to commit a `docs/queue/*.md` file in the repo to direct cc.
- `/ping` shows the same fallback.
- `/drop` same.
- The autonomous cron loop works regardless (cc reads docs/queue/ + docs/inbox/ + docs/drops/ from disk on every tick).

With bindings:
- Mike controls cc from his phone, on a court between matches, without touching a terminal.
- Visitors can ping/drop without git access.
- `/sprint?action=list` returns real entries that cc reads on the next tick.
