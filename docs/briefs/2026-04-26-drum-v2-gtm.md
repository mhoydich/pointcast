## Why this matters

The Drum Room is PointCast's most playful surface and the only one where multiple visitors can hear each other in real time. v2 adds the four levers a public launch needs: a leaderboard (status), a Tezos wallet connection point (identity + value), a pentatonic music engine (so collaboration sounds nice instead of chaotic), and a DRUM token accumulator (a clear future payoff). Together they turn a one-off curiosity into a daily-return loop with on-chain stakes.

If we ship the GTM below well, this is the first PointCast page that has a real shot at landing on Hacker News, getting cast on Farcaster, and bringing first-time visitors who arrive *because of /drum-v2*, not because they already knew about PointCast.

## Audience growth funnels

```
[discovery channel] → /drum-v2?via=X
    → first tap (badges, voice chip lights up)
        → ~30s in: name themselves + drum a few combos
            → wallet connect (Tezos) — converts to identity
                → leaderboard rank visible → competitive hook
                    → return visit → /drum-v2 → "you're #14 today"
                        → DRUM token claim (Phase C) → on-chain transfer event
```

**Conversion gates (where most drop-off happens):**

1. **First tap → 30s engagement** — make sure the first 5 seconds are immediately rewarding (combo +1 glow, voice chip flash, badge unlock toast). Already shipped.
2. **30s → name themselves** — currently a small input below the fold. Consider promoting it to the voice chip line.
3. **Name → wallet connect** — wallet adds noticeable value: leaderboard shows wallet address instead of "noun #42". When Phase C ships, they'll be airdropped DRUM. Hint at this in the wallet strip.
4. **Wallet → return** — daily streak, weekly leaderboard reset.

Metrics to instrument:
- `/api/visit` already tags the source — extend to log `?via=` param.
- `pc:drum:firstTapAt` localStorage timestamp.
- `pc:drum:walletConnectedAt` timestamp.
- `pc:drum:lastReturnAt` timestamp.

## Leaderboard

**Shipped:** v2 page now renders top 10 from `/api/drum/top` with rank colors (gold/silver/bronze for top 3), pixel noun avatars, count, and a "your rank" ribbon if you're not in the top 10. Wallet-connected players show their address instead of a noun number.

## Tezos wallet auth for participation

**Shipped:** The page reads `pc:wallets` / `pc:wallet-active` from localStorage and listens for the `pc:wallet-change` event fired by `WalletChip`. Wallet strip appears, claim pill changes to "PHASE C → YOU", leaderboard self-row shows wallet address.

**What's needed for "auth-gated participation" (next PR):**

1. **Server-side identity binding** — new endpoint `/api/drum/identify` that accepts `{ sessionId, walletAddress, signature }` where signature is a Beacon-signed message (`drum:claim-identity:<sessionId>:<timestamp>`). Verify the signature server-side. Store `wallet:<addr>` → `sessionId` in KV.

2. **Migrate count on identity binding** — when a wallet is bound for the first time, transfer that session's count and leaderboard entry to a wallet-keyed entry. Subsequent sessions from the same wallet (different browser, mobile) will all add to the same total.

3. **Soft-gate features** behind wallet:
   - **Don't gate** the basic drum tap — that's the funnel-top, must stay frictionless.
   - **Do gate** the "named on the leaderboard with my address" feature.
   - **Do gate** the eventual DRUM claim (Phase C — already gated by definition).
   - **Optional gate**: "DRUM Boost" — wallet-connected players earn 2× drums for the first 100 daily taps.

4. **Manus handoff for testing** — the actual Beacon picker flow (Kukai / Temple / Umami / AirGap) needs real-device testing. We can't validate the popup behavior from CLI.
