## What this means

- The frontend in `/bath` saves to `localStorage['pc:bath:saves']` *before* it POSTs, so users can keep their local saved tracks regardless of KV state.
- `/profile` rotation lane reads from localStorage as well.
- The only thing currently broken by `kv-not-bound`: cross-device sync (saving on phone, seeing it on laptop) and the planned `/bath/recent` global feed (D2-2).

## Recommendation

Provision `PC_BATH_KV` when convenient. Two paths:

### CLI (preferred)
```bash
npx wrangler kv namespace create "PC_BATH_KV"
# wrangler returns an `id`. Paste it into wrangler.toml:
```

```toml
[[kv_namespaces]]
binding = "PC_BATH_KV"
id = "<paste-from-wrangler>"
```

Commit + redeploy. The endpoint flips to `ok: true` automatically and starts persisting saves.

### Cloudflare Dashboard
1. CF → Workers & Pages → pointcast → Settings → Functions
2. KV bindings → Add → variable: `PC_BATH_KV` → create new namespace `pointcast-bath`
3. Save → trigger a redeploy

## What lights up after KV is bound

- `/profile` rotation lane → cross-device (saves on phone show up on laptop, both via the same wallet address)
- `/bath/recent` (D2-2 sprint item) → public global feed of recent saves
- POST `/api/bath` returns `{ ok: true, saved: {...} }` instead of `kv-not-bound`
- GET `/api/bath?wallet=tz1…` returns saves array
