## Design decisions worth recording

- **Mileage is approximate, and that's honest.** Used El Segundo Rec Park as the reference origin; each station's mileage is rounded to whole miles. Not a routing API, not a geocoded dataset — an editorial map. Same spirit as the name-drops list: "these are the places that matter," not "here's a GPS coordinate database."
- **Cardinal-direction chip as the compass.** A 24px mono badge with direction (N/NE/E/etc.). Lets the eye group stations geographically at a glance without a real map. The /tv STATIONS mode will inherit this taxonomy.
- **Loose substring match for in-range filter.** `SOCAL_TOKENS` list checks `meta.location.toLowerCase().includes(token)`. False positives are fine — the list is editorial, not legal. "Los Angeles" matches "Los Angeles County" correctly; that's the intent.
- **Palm Springs at 104mi gets included.** Deliberately outside the strict 100-mile line. Editorial call: the desert is part of the SoCal tonal register; North SD County at 99mi is included too. The "100 mile" framing is a direction more than a hard boundary.
- **No weather/sports on /local.** MorningBrief already covers that. Shipping them here would duplicate. /local is the *directory* layer; MorningBrief is the *today* layer. Separation of concerns.
- **Named-drop data is hardcoded.** Not authored as a collection because 7 items is too small to deserve its own collection schema. If it grows past ~15, promote to a collection + add a /name-drops page. Until then, inline.
- **`Place` JSON-LD with GeoCircle.** Agents can query the geometry programmatically — supports the future "which PointCast entities are in my range" query pattern without needing a separate geo endpoint.

## What didn't

- **Real geolocation.** No browser geolocation prompt. cc floor: don't ask for location without a user gesture + editorial context. A future /local upgrade can offer "center on my location" as an opt-in, defaulting to El Segundo.
- **Weather-grid across the radius.** Considered adding a mini weather chip per station. Deferred — Open-Meteo calls × 15 stations = 15 API calls per visit, not worth it at this stage. Better to render on demand when a visitor opens a specific station page.
- **Per-station pages.** Each station currently has no page. They're listed for now; the pages get authored when content for each accumulates.
- **Sunrise/sunset.** Natural fit for /local but adds one more API call and MorningBrief doesn't surface it either. Deferred.
- **Event aggregation.** "Events tonight" in the radius — flagged in 0282 — is the biggest standalone feature and deserves its own tick(s). Deferred.
