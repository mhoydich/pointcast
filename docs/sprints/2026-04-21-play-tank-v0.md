### Why this shape

Three decisions that collapsed the 3-day estimate into one session:

1. **Deterministic client-side positions.** Server holds the fish roster + event log + ecosystem numbers; positions are computed client-side from `fishPosition(fishId, elapsedMs, darting)`. Every client computes the same Lissajous curve for the same fishId. No 10Hz server push needed. No WebSocket. 1.5s poll is plenty.

2. **Polling over WebSocket for v0.** v0 state is ambient; 1.5s staleness is invisible to a human eye watching fish swim. If multi-visitor density exceeds ~20 concurrent, upgrade to a real WebSocket in v0.2. For now, `/api/tank/state` is cheap, cacheable, and doesn't require Hibernating Durable Objects.

3. **No blockchain in v0.** DRUM, Prize Cast, FA2 FishNouns all named in the brief's downstream specs, none ship here. Rhythm points aren't even tracked server-side in v0 — rate limits carry the full weight of abuse prevention.
