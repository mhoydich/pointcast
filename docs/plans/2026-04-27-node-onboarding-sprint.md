# Sprint 376 — node onboarding loop

Date: 2026-04-27  
Owner: Codex with Mike  
Status: launched

## Intent

Turn the new remote internship post, profile v3 audit, Mike's Spotify source, and the Nouns/collectibles surface into a coherent onboarding loop for the next builder.

The sprint should answer one practical question: if a curious person lands on PointCast today, can they understand what a node is, pick a lane, follow examples, and ship a first useful object?

## What Ships First

1. Publish this sprint plan as an internal operating note.
2. Publish a public sprint post at `/posts/pointcast-node-onboarding-sprint/`.
3. Publish Block 0376 so the plan is visible in the feed and agent-readable surfaces.
4. Keep `/profile` as the live source audit, now including Mike's Spotify and Nouns anchors.

## Workstream A — Node Starter Kit

Goal: make it obvious how an intern or collaborator starts a PointCast node.

- Create a small node-starter checklist.
- Name the first five starter node shapes:
  - Culture map
  - Sustainability watch
  - Art shelf
  - Science field note
  - Sport tracker
- Give each starter one first action, one data source, one public page idea, and one JSON/agent surface idea.
- Route candidates: `/for-nodes`, `/builders`, `/posts/pointcast-remote-internship/`.

## Workstream B — Nouns + Collectibles Trail

Goal: make the collectible layer feel inspectable, playful, and usable as identity.

- Keep Visit Nouns as the canonical Nouns collection entry.
- Use Noun 375 as the internship/node seed.
- Keep Noun 137, 888, 1111, 420, and 808 as profile v3 anchors.
- Add future blocks for collectible decision points only when they create a real action.
- Route candidates: `/profile`, `/collection/visit-nouns`, `/wallet`, `/market`.

## Workstream C — Spotify + Listening Nodes

Goal: turn Mike's Spotify profile from a link into a node ingredient.

- Treat listening as a source lane, not decoration.
- Start with one playlist/listening-node concept.
- Connect Spotify trails to mood, place, time, and blocks.
- Route candidates: `/profile`, `/drop`, `/drum-v3`, `/api/soundtracks.jsonl`.

## Workstream D — Recruitment Surface

Goal: make the internship post easier to act on.

- Keep the post concise enough to send directly.
- Add examples of first-node submissions.
- Add a small "raise your hand" packet if the volume becomes real.
- Use the AI collaborators as part of the offer: research, QA, drafting, coding, and taste calibration.

## Definition Of Done

- A public block exists.
- A public post exists.
- The build passes locally.
- The commit is pushed to `main`.
- The next builder can follow the trail without needing Mike to explain the whole thing in chat.
