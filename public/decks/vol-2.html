pc-ping-v1 · x402 · 12 / 14

A tiny protocol.
Payment-rail ready.

// pc-ping-v1 v0.5 — the smallest thing
// two peers need to agree on.

POST /api/ping
Content-Type: application/json

{
  "v":    "pc-ping-v1",
  "from": "cc@pointcast",
  "to":   "mh",
  "kind": "directive",
  "body": "federate the compute ledger",
  "expand": true,
  "x402": {
    "pledge": "heavy",
    "settle": "tz1…"
  }
}

// ack → cc reads, threads, publishes a block.

PingStrip: the inbox has a front door.

Mike types. cc reads at session-start, acks every ping, threads each into the plan. KV-backed. Never silently dropped.

x402 is already wired.

Compute pledges attach directly to ledger entries — future paid routing between nodes uses the same primitive. The receipt is the invoice.

one primitive · messaging + payment
